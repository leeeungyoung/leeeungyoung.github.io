문서 관리자 깃 & 깃허브 관련 명령어

깃 기본

  1. 깃 환경설정

    git config --global user.name "깃허브계정이름"
    git config --global user.email "깃허브계정주소"

    - 환경설정 제대로 입력됐는지 확인하기 : git config --list

  2. 깃 초기화

    git init

  3. 스테이징하기

    git add "스테이징할 파일명"

  4. 커밋하기

    git commit -m "메세지"

    - 한 번이라도 커밋했으면 스테이징과 커밋을 한 번에 하기 : git commit -am "메세지"

    - 커밋 확인하기 : git log --oneline

    - 깃의 상태 확인하기 : git status

    - 지정한 커밋 해시 이동하기 : git chechout 커밋해시

    - 가장 최근 커밋 취소하기(스테이지로) : git reset HEAD^

  5. 새 브랜치 만들기

    git branch 브랜치이름

    - 브랜치 이동하기 : git checkout 브랜치이름

    ★ 브랜치 위치 확인하기 ★ : git branch

    - 브랜치의 커밋 상태 그래프로 보기 : git log --oneline --branch --graph

    - master 브랜치에 다른 브랜치 병합하기 : git merge 병합할 브랜치 이름

    - 병합이 끝난 브랜치 삭제하기 : git branch -d 삭제할 브랜치 이름

    ★생성한 브랜치를 깃허브에 푸쉬★ : git push --set-upstream origin 브랜치명

깃 허브 관련

  1. 깃허브 원격저장소에 연결하기

    git remote add origin 복사한 리포지토리주소

    - 원격저장소와 지역저장소의 연결 확인하기 : git remote -v

    - 원격저장소에 파일 올리기(푸시하기) : git push -u origin master(-u는 처음 한 번만 사용)

    - 원격저장소에 파일 내려받기(풀하기) : git pull origin master

    - 잘못 연결된 깃허브 끊기 : git remote remove origin

  2. 깃허브 협업하기

    팀장이 깃허브의 기본을 셋팅하고 승인된 공동작업자에게만 커밋할 수 있는 권한을 부여한다.

    - 깃허브에서 셋팅하기

    1. 협업하기 위한 저장소 화면에서 [Setting]를 누른다.
    2. 좌측 메뉴의 [Collaborators]를 선택한다.
    3. [Add People]를 누른다.
    4. 텍스트 필드에 팀원의 깃허브 아이디나 메일 주소를 입력한 후 버튼을 누른다.
    5. 팀원은 협업자로 초대된 메세지 화면(오른쪽 상단 메뉴-Your organizations- Join-accept)에서 수락한다.
    6. 각 작업자의 컴퓨터에 지역저장소를 만든다.
    7. 각 작업자의 지역저장소와 기본 사용자 환경은 팀장의 환경과 동일하게 설정한다.

  3. 협업을 위한 원격저장소 복제하기

    git clone 복사한 리포지토리주소 저장할 디렉터리
